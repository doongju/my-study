# **객체 지향 4대 특징)** 캡슐화, 상속, 다형성, 추상화)

1. 캡슐화
- 데이터(변수)와 기능(메서드)을 하나의 알약(클래스)에 꽁꽁 싸매고, 내부가 안 보이게 숨기는 것 (**정보 은닉**).
- 외부에 있는 엉뚱한 코드가 내 중요한 데이터를 마음대로 바꾸는 걸 막기 위해서 사용하고 . 내부 구현을 숨기면 코드를 고칠 때도 외부 영향을 안 받아서 유지보수가 엄청 편해지는 장점이있다.

2. 상속

- 부모 클래스가 가진 기능을 자식 클래스가 그대로 물려받아 쓰는 것.

- 공통된 특징을 부모에 딱 한 번만 만들어 두고 재사용하면, 코드 양도 줄고 공통 수정도 한 번에 끝낼 수 있어.다.

3. 다형성
- 하나의 이름이나 기호가 **상황에 따라 다르게 동작**하는 능력.

- 객체를 쓰는 입장에서 정확히 어떤 종류인지 일일이 신경 쓰지 않고, 똑같은 명령어로 제어할 수 있게 만들어서 유연성을 극대화하기 하는 장점이있다.

4. 추상화
- 복잡한 내부 세계에서 **핵심적인 개념이나 특징만 뽑아내어** 단순하게 표현하는 것.

- 디테일한 구현은 나중으로 미루고, 큰 틀(설계도)을 먼저 짜고 복잡성을 줄여줘서 대규모 프로젝트를 할 때 좀더 틀을 잡고 만들수있는 장점이있다.
